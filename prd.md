# PRD: 사내 익명 피드백 시스템 (AnonVoice)

## 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | AnonVoice |
| 한 줄 설명 | 직원이 익명으로 팀/조직에 피드백을 남기고, 관리자가 대시보드로 확인하는 사내 서비스 |
| 핵심 가치 | 심리적 안전감 기반의 솔직한 피드백 문화 조성 |

---

## 타겟 유저 및 사용자 시나리오

### 유저 1: 일반 직원
- 팀 또는 조직 단위로 익명 피드백 작성
- 카테고리(문화, 프로세스, 리더십, 기타) 선택
- 이전에 작성한 피드백의 답변 확인 (익명 유지)

### 유저 2: 팀 리더/관리자
- 자기 팀에 달린 피드백 모아보기 대시보드
- 피드백에 익명 답변 달기
- 월별 감정 트렌드 리포트 확인

### 유저 3: HR 관리자
- 전사 피드백 통계 대시보드
- 위험 키워드 감지 알림 (예: "퇴사", "번아웃")
- 분기별 리포트 자동 생성

---

## 핵심 기능 명세

### 1. 익명 피드백 작성
- 카테고리 선택 (문화/프로세스/리더십/기타)
- 대상 팀 선택 (본인 팀 기본 선택, 다른 팀도 가능)
- 본문 작성 (최소 10자, 최대 1000자)
- 감정 태그 선택 (긍정/중립/부정)
- 제출 후 수정/삭제 불가 (익명성 보장)

### 2. 피드백 대시보드 (관리자용)
- 팀별 피드백 목록 (최신순/카테고리별 필터)
- 감정 분포 차트 (긍정/중립/부정 비율)
- 월별 트렌드 그래프
- 키워드 워드클라우드

### 3. 익명 대화 시스템
- 관리자가 피드백에 답변 작성
- 작성자가 익명으로 후속 코멘트 작성
- 쓰레드 형태의 대화 (최대 5회 왕복)

### 4. 알림 시스템
- 새 피드백 알림 (관리자에게 이메일)
- 답변 알림 (작성자에게 인앱 알림)
- 위험 키워드 감지 시 HR에게 즉시 알림

### 5. 통계 및 리포트
- 분기별 자동 리포트 생성 (PDF)
- 팀 간 비교 차트
- 참여율 통계

## 기술 스택

| 구분 | 기술 |
|---|---|
| 프론트엔드 | React 18 + Next.js 14 (App Router) |
| 상태 관리 | Zustand |
| UI 라이브러리 | Tailwind CSS + shadcn/ui |
| 백엔드 | Node.js + Express |
| 데이터베이스 | PostgreSQL 16 |
| ORM | Prisma |
| 인증 | NextAuth.js (사내 SSO 연동) |
| 배포 | Vercel (프론트) + Kubernetes (백엔드) |
| 모니터링 | Sentry + Prometheus |

---

## 주요 화면 구성

1. **로그인 페이지** - SSO 로그인 버튼
2. **피드백 작성 페이지** - 카테고리/팀 선택 + 본문 입력 폼
3. **내 피드백 페이지** - 작성한 피드백 목록 + 답변 확인
4. **관리자 대시보드** - 통계 차트 + 피드백 목록 + 답변 기능
5. **HR 대시보드** - 전사 통계 + 위험 알림 + 리포트

---

## 제약사항 및 보안 요구사항

- **익명성 보장이 최우선**: 피드백과 작성자 매핑 정보를 서버에 저장하지 않음
- **익명 대화 매핑**: 작성 시 1회용 익명 토큰 발급, 클라이언트에만 저장
- **데이터 보관**: 피드백 데이터 2년 보관 후 자동 삭제
- **접근 통제**: 관리자는 자기 팀 피드백만 열람 가능
- **감사 로그**: 관리자의 피드백 조회/답변 기록은 별도 로깅
- **HTTPS 필수**, 모든 API는 인증 토큰 필요
